# 영수증 정리함 (demo-receipt-box)

영수증 사진을 끌어다 놓으면 분류·집계되는 앱. **사진도 내역도 내 기기에만** 저장된다.

- 사진 드래그·촬영·앨범 선택으로 등록 (여러 장 한 번에) — 증빙 사진이 항목마다 붙어 있는 정리함
- 날짜·상호·금액·분류(식비/교통/사무용품/접대·경조/기타)·메모를 10초 입력
- 월별 집계(분류별 막대) + **CSV 내보내기**(엑셀 호환) — 경비 청구·증빙 정리에
- 설치 없이 홈 화면에 추가, 오프라인 동작. 서버 전송 0

> 이 앱은 **"회사원도 이런 앱을 만들 수 있다"** 교육용 데모 시리즈의 하나입니다.
> 원형은 실제 교육 과정의 '영수증 지침' 실습 과제 — 실습 한 조각이 완성 제품이 되는 과정을 보여줍니다.

## 🙋 이 앱, 나도 만들 수 있을까?

**네, 코딩을 몰라도 됩니다.** 이 앱도 AI(Claude)에게 말로 시켜서 만들었습니다.

### 따라 만들기 — 이 프롬프트를 그대로 써보세요

```
영수증 정리 웹앱을 만들어줘.

- 영수증 사진을 드래그하거나 폰 카메라로 찍어서 등록
- 사진마다 날짜/상호/금액/분류(식비·교통·사무용품·접대·기타)/메모를 입력
- 월별로 분류별 합계를 막대로 보여주고, CSV로 내보낼 수 있게(엑셀 호환)
- 사진과 데이터는 서버 없이 전부 브라우저에만 저장
  (사진은 용량이 크니 localStorage 말고 IndexedDB에)
- 어두운 배경, 모바일 우선
```

**포인트**: 사진처럼 큰 데이터는 localStorage(5MB 한계)가 아니라 **IndexedDB**에 — 이런 선택도 AI에게 "사진 저장은 어디에 하는 게 맞아?"라고 물으면 알려줍니다.

### 이 앱에 "AI 자동 인식"이 없는 이유 — 실패도 교재입니다

처음엔 온디바이스 AI(Gemini Nano)로 사진에서 금액을 자동 추출하는 기능을 넣었습니다. 그런데 **실제 편의점 영수증으로 검증하니 2,000원을 40,190원으로 읽었습니다** — 소형 온디바이스 모델은 감열지 영수증의 숫자 정밀 인식(OCR)에 아직 못 미칩니다. 금액이 틀리는 자동화는 없느니만 못하므로 **기능을 뺐습니다.**

여러분이 만들 때도 같은 순서를 권합니다: **만들기 → 실제 데이터로 검증 → 기준 미달이면 과감히 빼기.** AI에게 시켜서 기능을 넣는 것만큼, 검증해서 빼는 것도 실력입니다.

### 더 쉬운 방법 — 이 저장소를 통째로 보여주기

```
이 앱 구조 그대로 "명함 정리함"으로 바꿔줘.
명함 사진 → 이름/회사/연락처 정리, CSV 내보내기.
```

## 📚 알아두면 좋은 개념 (비개발자용 최소 지식)

| 개념 | 한 줄 설명 |
|---|---|
| **IndexedDB** | 브라우저 안의 큰 저장소 — 사진·파일도 담을 수 있는 localStorage의 형 |
| **OCR** | 사진 속 글자를 텍스트로 읽는 기술 — 정밀도가 생명이라 검증 없이 믿으면 안 됨 |
| **검증 우선** | 기능은 실제 데이터로 판정하고, 기준 미달이면 빼는 것 — 이 앱이 그렇게 만들어짐 |
| **PWA** | "홈 화면에 추가"하면 앱처럼 아이콘·전체화면·오프라인 실행이 되는 웹앱 |
| **CSV** | 엑셀이 바로 여는 표 텍스트 파일 — 시스템 간 데이터 주고받기의 공용어 |
| **배포** | 내 컴퓨터의 앱을 인터넷 주소로 여는 것. 이 시리즈는 Vercel(무료)을 씁니다 |

4. **이 저장소의 [CLAUDE.md](CLAUDE.md)** — 앱 구조와 동작 원리 정리. AI에게 읽혀도 됩니다.
